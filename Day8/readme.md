<h2>Conditional Statements</h2> <p>Conditional statements are used to perform operations based on conditions.</p> <h3>Types of Conditional Statements</h3> <h4>1. If – Else</h4> <p>Checks a condition and executes code if the condition is true.</p> <pre> stock = -20 is_available = false if stock > 0 is_available = true else puts "Item not available" end </pre>
<h4>2. Else If (elsif)</h4> <p>Used when multiple conditions need to be checked.</p> <pre> price = 350 discount = 0 if price > 100 discount = 10 elsif price > 200 discount = 20 elsif price > 300 discount = 30 elsif price > 400 discount = 40 else discount = 5 end puts discount </pre>
<h4>3. Ternary Operator</h4> <p>Short form of if-else written in a single line.</p> <pre> puts is_active ? "Item present" : "Not present" </pre>
<h4>4. Unless</h4> <p>Opposite of <b>if</b>. Executes when condition is false.</p> <pre> stock = 15 unless stock < 0 puts "Stock available" else puts "Stock not available" end </pre>
<h4>5. Case Statement</h4> <p>Used to match multiple conditions similar to switch-case.</p> <pre> price = 350 discount = 0 case when price >= 100 && price < 200 discount = 10 when price >= 200 && price < 300 discount = 20 when price >= 300 && price < 400 discount = 30 when price >= 400 && price < 500 discount = 40 else discount = 5 end </pre> <p><b>Using ranges inside case:</b></p> <pre> case price when 100..200 discount = 10 when 200..300 discount = 20 when 300..400 discount = 30 when 400..500 discount = 40 else discount = 5 end puts discount </pre> <hr> <h2>Access Modifiers</h2> <p>Access modifiers control the visibility of methods and classes.</p> <h3>Types of Access Modifiers</h3> <ul> <li><b>Public</b> – Methods can be accessed from anywhere</li> <li><b>Private</b> – Methods can be accessed only inside the same class</li> <li><b>Protected</b> – Methods can be accessed inside the class and its subclasses</li> </ul> <h3>Important Notes</h3> <ul> <li>Private methods cannot be called directly outside the class</li> <li>They can be accessed indirectly using a public method</li> <li>Protected methods can be accessed in child classes</li> </ul>
